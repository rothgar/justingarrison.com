## Description

Like many people experimenting with self-hosting LLMs Justin tried it on the hardware he already had. This worked, but had limitations on capacity and access. The next step was to get dedicated hardware, but running multiple models, sharing, and deploying agents still had too many constraints. Then Justin started deploying LLMs to Kubernetes using substrate from Google. This talk reviews what substrate is, what problems it solves, and how it compares to other options.
